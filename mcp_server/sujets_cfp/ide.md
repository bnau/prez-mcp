# Copilot, Cursor & cie : Explorons le vaste monde des assistants de code boostés par IA

En 2026, beaucoup de développeurs ont un assistant IA dans leur éditeur. Certains s’en servent pour accélérer leurs tâches répétitives, d’autres ne peuvent plus coder sans lui. Quand nous avons demandé à notre entreprise de nous équiper, on nous a naturellement répondu oui, mais lequel choisir ?

C’est là que les ennuis ont commencé. Copilot, Junie, Cursor, Claude Code, Gemini CLI, Continue, Zed, Windsurf... L’écosystème est immense, les modèles évoluent sans cesse et les fonctionnalités disponibles varient autant que les prix. Et surtout : comment les comparer objectivement ?

Ce talk est le récit de cette exploration.

Je partagerai les critères concrets utilisés pour évaluer les différents assistants de code, les résultats obtenus et les difficultés rencontrées en chemin. Je m'appuierai sur des démos, des cas d’usage réels et resterai les plus factuels possible, tout en signalant quand un point relève de la subjectivité.

L’objectif n’est pas de promouvoir un assistant particulier, mais de donner une grille de lecture claire pour aider chacun à choisir la solution qui lui conviendra le mieux.


## Références

La présentation abordera les points suivants :

* Introduction des types d'assistants (orientés CLI, intégrés dans l'IDE, fournis par l'IDE)
* Compatibilité dans les différents éco-systèmes
* Support de rules, hooks et autres customisations
* Pricing et consommation de tokens
* Inférence locale
* Propriété du code source

Il y aura aussi deux démos, très courtes de Claude Code pour la CLI et Copilot pour l'intégration dans l'ide

Les modèles seront très peu comparés, les résultats sont très subjectifs et similaires.
